# Robotics Autonomous Systems Case Study

A public case study about my work building autonomous and driver-control systems for competitive VEX robotics.

## Overview

As lead programmer for Heroic Robotics, I worked on C++ autonomous systems, odometry, sensor feedback, route testing, and driver-control logic.

This repo is a cleaned public case study. It does not include private team code or active competitive strategy.

## Problem

VEX autonomous programming requires fast, accurate, and repeatable movement. Small issues with drivetrain tuning, sensor reliability, battery level, field friction, and route timing can cause major problems during matches.

## My Role

- Lead programmer
- Driver
- Notebook contributor
- Autonomous testing and strategy support

## Areas I Worked On

- C++ robot control
- Odometry-based movement
- Sensor-assisted corrections
- Autonomous route testing
- Driver-control improvements
- Engineering notebook documentation
- Competition strategy

## Results

This work contributed to state qualifications, awards, tournament success, and qualification for VEX Worlds.

## Status

Public case study. Full implementation and current competitive code are private.
